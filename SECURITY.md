# Security Policy

## Supported Versions

The following versions of **RedPixelDream** are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.6.x   | :white_check_mark: |
| < 0.6   | :x:                |

## Reporting a Vulnerability

I take the security of this project seriously. If you find a security vulnerability, please do not disclose it publicly. Instead, please report it using one of the following methods:

1. **GitHub Security Advisory**: You can report a vulnerability directly via the "Security" tab of this repository.
2. **Issue Tracker**: If the vulnerability is not sensitive, you can open a standard issue.

### What to include in a report?
A good security report should include:
- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact.
- Any suggested fixes (optional).

## Disclosure Policy
When a vulnerability is reported, I will:
- Acknowledge the receipt of the report as soon as possible.
- Work on a fix and release a new version.
- Provide credit to the researcher (if desired).

Thank you for helping keep RedPixelDream secure!
